Engine de jogos em ASCII, feita na linguagem C
A engine foi feita para se estudar as diversas partes que compõem um jogo de computador. Embora seja muito simples, a lógica de funcionamento do projeto é semelhante à dos jogos e engines "reais", servindo assim para o aprendizado de tais técnicas utilizando ferramentas familiares aos recém-ingressados a um curso de computação, no caso a linguagem C e suas bibliotecas, tanto as padrões como as mais usuais.

A engine funciona tanto em Windows quanto em (GNU/)Linux utilizando quase completamente o mesmo código, sendo as únicas diferenças:
  • a função de limpar a tela(usando a API do Windows, e usando o comando clear do terminal do Linux)
  • a entrada do teclado(utilizando a conio no Windows; ainda em construção no Linux)
Para ambos, o compilador utilizado foi o gcc, sendo a versão 4.4.0 (mingw) no caso do Windows e 4.3.3 (Ubuntu) no caso do Linux (andLinux).


Organização do código:

O código da engine é compilado em uma biblioteca de link dinâmico, uma dll no caso do Windows, e um arquivo so(shared object) no caso do Linux, agilizando a compilação dos jogos. Além disso, a biblioteca pode ser acessada por outras linguagens, aproveitando a agilidade proporcionada pelo código compilado em C.

No atual estágio do desenvolvimento,  os jogos estão localizados em pastas dentro do próprio diretório da engine. Isso se deve apenas ao contínuo desenvolvimento em ambas as partes, que ocorre paralelamente, agilizando adições na engine para suprir as necessidades que surgem nos jogos.


Como compilar:

Como dito anteriormente, o compilador utilizado foi o gcc.

Boa parte das modernas distribuições de Linux já o incluem, e se não for o caso, ele pode ser instalado usando o sistema de pacotes da distribuição.

No caso do Windows, deve ser instalado o pacote MinGW(Minimalist GNU for Windows), que consiste nos compiladores gcc/g++/etc e mais outras ferramentas como o make.

Depois de instalado, é necessário adicionar a pasta que contém o executável do compilador na variável PATH do Windows, permitindo que ele seja executado de qualquer pasta. Tal procedimento varia de acordo com a versão do Windows.

O código dispõe de diversos scripts que automatizam o processo de compilação:
Cada pasta de projeto contém seus próprios scripts, em suas versões para Windows e Linux, entre eles:
  • compilar, que compila o projeto já linkando com a engine. O executável é colocado na pasta bin.
  • testar, que executa o arquivo
  • debugar, que roda o programa no gdb, para procurar falhas de memória, entre outros bugs
  • profile, que compila o arquivo com a flag de profile, mostrando estatísticas de execução
  • lib_compile, que compila a engine, chamando seu próprio script
  • sprites_copy, que copia os sprites do jogo para a pasta dos binários
No linux, há ainda o instalar_so.sh, que copia a biblioteca para o sistema(embora ele possa ser usado de modo semelhante à dll; para isso, é necessário alterar o parametros.sh).
Além disso, existem os seguintes scripts auxiliares:
  • nome, que define o nome do projeto, sendo usado para acessar a pasta e nomear o aplicativo
  • parametros, que contem os parametros de compilação comuns a alguns scripts. É nele que você deve colocar as flags de compilação para utilizar alguma biblioteca(ex: -lm, para math.h)

Nas pastas de script da engine, existem os scripts que servem para a sua compilação. Entretanto, eles não precisam ser chamados diretamente, já que existem o lib_compile e instalar_so.sh nas pastas de cada projeto.


Tipos de dados definidos:

//--Sprite--//
typedef struct _Sprite
{   
    // "imagem" do sprite(ponteiro pra primeira linha da lista)
    Linha *img;
   
    // Ponteiro pro proximo sprite da lista ligada
    struct _sprite *prox;
} Sprite;

O tipo central da engine, cada sprite consiste em uma struct que contém uma lista ligada de strings, que são cada linha da imagem. Devido a essa implementação, os sprites podem possuir variados números de linhas, cada uma com diferentes tamanhos:

//--struct com cada linha da imagem--//
typedef struct _Linha
{
    // String da linha da imagem
    char *str;
   
    // Ponteiro pra proxima linha da lista ligada
    struct _Linha *prox;
} Linha;

Além disso, cada sprite possui um ponteiro para um eventual próximo sprite, para o caso de se criar uma lista ligada de sprites, formando uma animação:

//--Animacao--//
typedef struct _Anima
{
    // Numero do sprite mostrado
    unsigned short int n;
   
    // Ponteiro pro primeiro sprite da lista
    Sprite *spr;
   
    // Ponteiro pra proxima animacao
    struct _Anima *prox;
} Anima;

Cada animação também possui um ponteiro para uma próxima animação, caso seja feita uma lista ligada delas(por exemplo, a primeira pode ser o personagem parado, a próxima ele andando, etc).
O código que lida com as animações ainda não foi implementado.

Funções disponibilizadas pela engine:
  • Funções de desenho, no draw.h:
void sprite_draw(sprite, int, int);
Desenha um sprite na tela, recebendo as coordenadas x e y da tela.
Caso parte do sprite esteja fora dos limites, essa parte é enviada para o outro extremo da tela.

void str_draw(char*, int, int);
Escreve uma string na tela.

void str_draw_v(char*, int, int);
Escreve uma string na tela, verticalmente.

void h_line_draw(char c, int y);
Desenha uma linha horizontal, na altura y, composta de caracteres c.

void le_string_txt(char*, char*);
Lê uma string de um arquivo texto. O primeiro argumento é a string de destino, e o segundo é o caminho para o arquivo.

void le_sprite_txt(sprite**, char*);
Lê um sprite de um arquivo texto. O segundo argumento é o caminho para o arquivo.

void free_sprite(sprite*);
Libera um sprite alocado dinamicamente. Deve ser utilizada para liberar um sprite lido de um txt.

  • Funções de leitura do teclado:
int up(void);
int down(void);
int left(void);
int right(void);
Funções que testam se as setas do teclado foram pressionadas. Retornam 0 se não.

int esc(void);
Testa se a tecla ESC foi pressionada. No exemplo, é usada para sair do jogo.

int espaco(void);
Testa se a barra de espaço foi pressionada. É utilizada para iniciar o jogo de pong.

A adição de mais dessas funções é muito simples, sendo realizada conforme for necessário.

Essas variáveis estão declaradas na engine, têm seu valor definido pelo código contido no arquivo vars.h de cada projeto, no início de sua execução.


Como criar um jogo com a engine:

Simples, tudo o que é necessário é copiar o projeto template e substituír tudo o que tiver o nome "template" pelo nome do seu jogo. Isso inclui a pasta, o arquivo fonte e a variável declarada nos scripts nome.(bat|sh). Depois disso, é só programar!
